DATABASE — Migrations (5 files)

    Columns: id, name (unique), icon, timestamps

  - 2024_01_01_000002_create_users_table.php

             expo_push_token, remember_token, timestamps

    Columns: id, user_id (FK→users), category_id (FK→categories), name,

             timestamps

  - 2024_01_01_000005_create_reservations_table.php
    Columns: id, buyer_id (FK→users), product_id (FK→products), quantity,
             status (enum: pendiente/confirmada/rechazada/completada),
             notes (text), timestamps

MODELS (5 files)
  - User.php — HasApiTokens, Notifiable, HasFactory
    Relations: location (hasOne), products (hasMany), reservationsAsBuyer (hasMany)
    Fillable: name, email, password, role, phone, avatar_url, expo_push_token

  - Product.php — HasFactory
    Relations: user (belongsTo), category (belongsTo), reservations (hasMany)
    Fillable: user_id, category_id, name, description, price, unit, stock, image_url, status

  - Category.php — HasFactory
    Relations: products (hasMany)

  - Location.php — HasFactory
    Relations: user (belongsTo)
    Fillable: user_id, commune, region, latitude, longitude

  - Reservation.php — HasFactory
    Relations: buyer (belongsTo User), product (belongsTo Product)
    Fillable: buyer_id, product_id, quantity, status, notes

CONTROLLERS (5 files)
  - AuthController.php
    register() — validates name, email (unique), password (min:8, confirmed), role
                creates user with hashed password, returns user + Sanctum token
    login() — validates credentials, returns user + token
    logout() — revokes current token
    me() — returns user with location

  - ProductController.php
    index() — paginated list with status='disponible', filters: category_id, search, commune
    show() — single product with relationships
    store() — validates fields, uploads image to Cloudinary, creates product
    update() — owner-only, updates fields, optional new image
    destroy() — owner-only delete

  - ReservationController.php
    index() — role-based: buyer sees own, farmer sees incoming
    store() — validates product disponibility + stock, creates reservation in
              DB transaction, decrements stock
    updateStatus() — farmer only, validates status transitions

  - CategoryController.php
    index() — returns categories with products_count

  - ProfileController.php
    show() — returns user with location
    update() — updates user + upserts location

TRAITS

=== Fixed Bugs in Existing Code ===

  1. Migration: reservations.buyer_id foreign key
     - Before: constrained() — inferred 'buyers' table (doesn't exist)
     - After: constrained('users') — correctly references users table

  2. Controllers: ApiResponse trait import
     - Before: use ApiResponse; inside class (resolved to App\Http\Controllers\ApiResponse)
     - After: Added use App\Traits\ApiResponse; import at top of all 5 controllers

  3. bootstrap/app.php: Sanctum middleware class name
     - Before: EnsureFrontendRequestsAreFromBackend (wrong class name)
     - After: EnsureFrontendRequestsAreStateful (correct class name)

  4. ReservationController::updateStatus — Stock management logic
     - Before: Decremented stock again on confirmada/completada (stock was already
       decremented when reservation was created)
     - After: Only restores stock on rechazada (increment); no change on confirm/completada

  5. ProfileController::update — expo_push_token missing
     - Before: Did not accept or save expo_push_token
     - After: Added expo_push_token to validated fields (nullable, string, max:200)
